# 前端页面设计 ✅
能够展示音标、译文、长难句分析、点击句子时展示该句子的核心词汇、词组、语法结构。

两种模式：
1. 阅读模式：展示原文、音标、译文、循环播放音频。
2. 学习模式：展示原文、音标、长难句分析（点击句子时展示该句子的核心词汇、词组、语法结构）。

音标展示：
上面是英文，下面是音标。英文和音标对齐。像中文和拼音对齐一样。
json中音标和英文是一整句的形式保存的，因此页面展示时需要通过空格切分单词和音标。

布局：
左右两侧面板，6:4。左侧是原文和音标，右侧是译文/长难句分析。

# 数据处理 (create_audio.py) 建议架构 ✅

既然音频文件是通过句子的 MD5 命名，这个脚本可以做到增量生成（也就是不重复生成已经存在的文件，节约时间和接口额度）：

脚本读取 article.json。
遍历所有段落和句子，提取 original_text。
计算该句子的 MD5 值，拼接出路径 data/audio/MD5.mp3。
判断该文件路径是否已存在：如果不存在，则调用 google TTS 语音合成接口，并保存为对应的 MP3。
