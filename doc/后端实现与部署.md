# 后端实现与部署（PHP 方案）

## 背景

服务器环境：PHP 8.2.29 ✅ + Apache mod_php ✅ + JSON 扩展 ✅

**PHP 方案的优势**：Apache 原生支持 PHP，不需要反向代理，不需要守护进程，部署最简单 —— 上传文件即可运行。

## 架构

```
Apache (80端口)
  └── /var/www/html/learn/
       ├── index.html          ← 前端入口
       ├── api/                ← PHP API
       │    └── index.php      ← 路由分发
       ├── .htaccess           ← URL 重写规则
       ├── data/procesed/*.json  ← 文章数据
       ├── data/audio/*.mp3      ← 音频文件
       ├── css/
       └── js/
```

> [!NOTE]
> 不再需要 ProxyPass 反向代理。前端直接请求 `/api/articles`，Apache 通过 `.htaccess` 重写到 `api/index.php`，PHP 直接返回 JSON。

## Proposed Changes

### PHP API

#### [NEW] [index.php](file:///Users/kooreywang/myproject/learn/api/index.php)

路由分发 + API 逻辑，替代 `server/app.js`：

- `GET /api/articles` → 扫描 `data/procesed/*.json`，返回 `[{id, title}]`
- `GET /api/articles/{article_id}` → 读取对应 JSON 文件，原样返回

---

### URL 重写

#### [NEW] [.htaccess](file:///Users/kooreywang/myproject/learn/.htaccess)

Apache URL 重写规则：
- `/api/articles` → `api/index.php?action=list`
- `/api/articles/{id}` → `api/index.php?action=detail&id={id}`
- 其他请求正常走静态文件

---

### 前端文件迁移

将 `server/static/` 下的前端文件移到项目根目录，使目录结构与服务器 DocumentRoot 对齐：

#### [MOVE] `server/static/index.html` → `index.html`
#### [MOVE] `server/static/css/` → `css/`
#### [MOVE] `server/static/js/` → `js/`

---

### Apache 配置

#### [MODIFY] Apache learn.conf

移除不再需要的 ProxyPass 行（`/api → 127.0.0.1:8000`），改由 `.htaccess` 处理路由。
